# Functional Requirements Document (FRD)

**Project:** AyurMantra Digital Platform
**System:** Dynamic Website + Patient Portal + Admin Management System
**Version:** 1.0

---

# 1. Introduction

## 1.1 Purpose

This document defines the **detailed functional requirements** for the AyurMantra digital platform. It describes the behavior of the system, the functions it must support, and how users interact with it.

The system will include:

1. Public Website (End-User Interface)
2. Patient Portal
3. Admin Management System (CMS + Clinic Operations)

The platform must be **fully dynamic**, allowing clinic staff to manage all website content, treatments, media, products, and patient data through an admin interface without requiring developer intervention.

---

# 2. System Overview

## 2.1 System Components

| Component      | Description                                        |
| -------------- | -------------------------------------------------- |
| Public Website | Informational site for visitors and patients       |
| Patient Portal | Secure login for patients                          |
| Admin Panel    | Clinic management dashboard                        |
| Database       | Stores all system data                             |
| API Layer      | Handles communication between frontend and backend |

# 3. User Roles and Permissions

## 3.1 Roles

### Visitor

Unregistered website users.

Permissions:

* Browse website
* View treatments
* Read blogs
* Submit contact form
* Book appointment

---

### Patient

Registered user with login access.

Permissions:

* Manage profile
* Book appointments
* View medical records
* Download prescriptions
* View treatment history

---

### Clinic Staff

Clinic operational staff.

Permissions:

* Manage appointments
* View patient details
* Upload prescriptions
* Update patient treatment data

---

### Doctor

Medical practitioner.

Permissions:

* View patient history
* Add prescriptions
* Upload treatment notes
* Manage consultation records

---

### Admin

System administrator.

Permissions:

* Full system control
* Manage content
* Manage users
* Manage services
* Manage products
* Manage reports
